# 🎮 Game Project

A collaborative game development project with full Scrum team structure.

## 📋 Team Structure

| Role | Responsibilities |
|------|------------------|
| **Product Owner** | Vision, backlog prioritization, stakeholder management |
| **Business Analyst** | Requirements gathering, user stories, acceptance criteria |
| **Engineer L** | Core development, feature implementation |
| **Senior Engineer** | Architecture, code review, technical decisions |
| **QA** | Testing, bug reports, quality assurance |
| **Scrum Master** | Process facilitation, removing blockers |

## 🚀 Quick Start

```bash
# Run the game
open index.html

# Or serve locally
python3 -m http.server 8080
```

## 📁 Project Structure

```
game-project/
├── docs/              # Documentation & specs
├── src/               # Source code
├── tests/             # Test files
├── backlog/           # Sprint & user stories
├── meetings/          # Meeting notes
└── index.html         # Main game entry
```

## 🔄 Development Workflow

1. Pick a task from `backlog/sprint-current.md`
2. Create a branch: `git checkout -b feature/your-feature`
3. Code & test
4. Run `./deploy.sh` to auto-push
